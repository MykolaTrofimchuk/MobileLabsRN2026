# FirstMobileApp - Лабораторна робота №1

## Опис проєкту
Цей проєкт є найпростішим мобільним додатком, створеним за допомогою **React Native** та фреймворку **Expo**.
Метою проєкту є ознайомлення з базовими компонентами React Native (View, Text, FlatList, ScrollView, TextInput, Image), а також налаштуванням середовища розробки.

У додатку реалізовано верхню вкладкову навігацію (Top Tabs Navigation) за допомогою `@react-navigation/material-top-tabs`, яка містить 3 екрани:
1. **Головна (Новини)** — список новин, реалізований через `FlatList`.
2. **Фотогалерея** — відображення сітки зображень.
3. **Профіль (Реєстрація)** — форма для введення даних користувача з використанням `TextInput`.

Додаток містить статичну шапку (Header) з логотипом університету та підвал (Footer) з інформацією про розробника (ПІБ та група).

## Інструкція із запуску

Для запуску проєкту локально виконайте наступні кроки:

1. Переконайтеся, що у вас встановлено **Node.js**.
2. Відкрийте термінал у папці з проєктом.
3. Встановіть необхідні залежності командою:
   ```bash
   npm install
4. Запустіть проєкт:
    ```bash
    npx expo start

## Основні способи запуску та тестування мобільного додатка
Під час розробки на React Native + Expo доступні кілька основних способів тестування додатка. Кожен з них має своє призначення та особливості:

#### 1. Фізичний пристрій (через додаток Expo Go)
   - Призначення: Тестування додатка на реальному "залізі", щоб перевірити, як він працює в реальних умовах.

   - Особливості: Не потребує підключення через кабель. Достатньо завантажити додаток Expo Go (з App Store або Google Play), підключити телефон та ПК до однієї Wi-Fi мережі і відсканувати QR-код з терміналу.

   - Відмінності: Найшвидший спосіб побачити реальну продуктивність додатка. Підтримує гаряче перезавантаження (Fast Refresh) — зміни в коді миттєво відображаються на екрані телефону.


#### 2. Віртуальний емулятор (Android Studio / iOS Simulator)
   - Призначення: Тестування додатка на різних моделях телефонів та різних версіях ОС без необхідності купувати ці пристрої фізично.

   - Особливості: Вимагає встановлення важкого ПЗ (Android Studio для Windows/Mac або Xcode тільки для Mac). Забирає значну частину оперативної пам'яті комп'ютера.

   - Відмінності: Дозволяє гнучко налаштовувати параметри телефону, імітувати зміну геолокації, рівень заряду батареї, повільний інтернет тощо. В емуляторі зручно робити скріншоти для звіту.


#### 3. Web-браузер
   - Призначення: Швидкий перегляд верстки та базового функціоналу без залучення мобільних пристроїв.

   - Особливості: Запускається натисканням клавіші w у терміналі Expo. Відкриває додаток як звичайну веб-сторінку.

   - Відмінності: Не всі мобільні бібліотеки ідеально працюють у вебі. Цей спосіб підходить лише для базової перевірки UI, але не годиться для тестування специфічних мобільних функцій (камера, пуш-сповіщення, системні відступи на зразок SafeArea).


#### 4. Онлайн-середовище Expo Snack
   - Призначення: Швидке прототипування, обмін кодом або виконання навчальних завдань.

   - Особливості: Не потребує встановлення Node.js та жодного ПЗ на комп'ютер. Код пишеться та компілюється безпосередньо у браузері (у хмарі).

   - Відмінності: Ідеально підходить для швидкого старту, але має обмеження для великих проєктів (менше контролю над файловою системою та налаштуваннями збірки).
